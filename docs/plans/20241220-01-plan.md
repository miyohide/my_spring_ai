# Spring AIを使ったBedrock連携REST API実装計画

## 計画日
2024年12月20日

## 実装目標
"/"にPOSTリクエストを送信し、クエリパラメータで指定されたメッセージをSpring AI経由でBedrockに送信し、応答をREST APIで返すシステムを実装する。

## 実装手順

### 1. プロジェクト設定
- `build.gradle`に必要な依存関係を追加
- `application.properties`にBedrock設定を追加

### 2. レスポンスDTO作成
- `BedrockResponse`クラスを作成
- 生成されたテキストを格納するフィールドを定義

### 3. Serviceクラス作成
- `BedrockService`クラスを作成
- `ChatClient`を注入してBedrock連携処理を実装

### 4. Controllerクラス作成
- `BedrockController`クラスを作成
- POSTエンドポイント実装
- クエリパラメータからメッセージを取得
- Service呼び出しとレスポンス返却

### 5. テストクラス作成
- `BedrockControllerTest`を作成
- 正常系・異常系のテストケースを実装

## 実装するファイル

1. `build.gradle` - 依存関係追加
2. `src/main/resources/application.properties` - 設定ファイル
3. `src/main/java/com/example/springaiapp/BedrockResponse.java` - レスポンスDTO
4. `src/main/java/com/example/springaiapp/service/BedrockService.java` - Serviceクラス
5. `src/main/java/com/example/springaiapp/controller/BedrockController.java` - Controllerクラス
6. `src/test/java/com/example/springaiapp/controller/BedrockControllerTest.java` - テストクラス

## 実装詳細

### BedrockResponse
- `response`フィールド（String型）
- コンストラクタとgetter

### BedrockService
- `generateResponse(String message)`メソッド
- ChatClientを使用してBedrock呼び出し

### BedrockController
- `@PostMapping("/")`
- `@RequestParam("message")`でパラメータ取得
- バリデーション実装

### テスト
- 正常系：メッセージ送信とレスポンス確認
- 異常系：パラメータ不正時のエラーハンドリング
