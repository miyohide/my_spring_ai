# Spring AIを使ったBedrock連携REST API実装調査（クエリパラメータ版）

## 調査日
2025年10月11日

## 調査目的
"/"にPOSTリクエストを送信した際に、Spring AIを使ってAmazon Bedrockにリクエストを送信し、応答をREST APIで返す実装方法を調査する。
Bedrockに送信するメッセージはPOSTリクエストのクエリパラメータで指定できるようにする。

## 調査結果

### 1. 必要な依存関係

```gradle
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    implementation 'org.springframework.ai:spring-ai-bedrock-ai-spring-boot-starter'
    implementation 'software.amazon.awssdk:bedrock-runtime'
}
```

### 2. 設定ファイル（application.properties）

```properties
spring.ai.bedrock.aws.region=ap-northeast-1
spring.ai.bedrock.aws.access-key=${AWS_ACCESS_KEY_ID}
spring.ai.bedrock.aws.secret-key=${AWS_SECRET_ACCESS_KEY}
spring.ai.bedrock.anthropic.claude.model=anthropic.claude-3-sonnet-20240229-v1:0
spring.ai.bedrock.anthropic.claude.options.temperature=0.7
spring.ai.bedrock.anthropic.claude.options.max-tokens=1000
```

### 3. 実装アーキテクチャ

#### Controller層
- POSTリクエストを受け取る
- クエリパラメータからメッセージ（プロンプト）を取得
- Serviceクラスに処理を委譲
- レスポンスを返す

#### Service層
- Spring AIのChatClientを使用
- Bedrockへのリクエスト送信
- レスポンスの処理

### 4. 実装に必要なクラス

#### BedrockController
- `@RestController`アノテーション
- `@PostMapping("/")`でルートパスのPOSTを処理
- `@RequestParam`でクエリパラメータからメッセージを取得
- レスポンス用DTOクラス

#### BedrockService
- `@Service`アノテーション
- `ChatClient`を注入
- Bedrockとの通信処理

#### レスポンスDTO
- レスポンス用：生成されたテキストを含む
- リクエストはクエリパラメータのため、DTOは不要

### 5. クエリパラメータの実装詳細

#### Controller実装例
```java
@PostMapping("/")
public ResponseEntity<BedrockResponse> generateText(
    @RequestParam("message") String message) {
    // messageパラメータを使用してBedrockにリクエスト
}
```

#### リクエスト例
```
POST /?message=こんにちは、元気ですか？
```

#### パラメータ検証
- `@RequestParam(required = true)`でパラメータ必須化
- `@Valid`と`@NotBlank`でバリデーション
- URLエンコーディング対応

### 6. Spring AIのChatClient使用方法

```java
@Autowired
private ChatClient chatClient;

public String generateResponse(String prompt) {
    return chatClient.prompt()
        .user(prompt)
        .call()
        .content();
}
```
