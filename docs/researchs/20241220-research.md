# Spring AIを使ったBedrock連携REST API実装調査

## 調査日
2024年12月20日

## 調査目的
"/"にPOSTリクエストを送信した際に、Spring AIを使ってAmazon Bedrockにリクエストを送信し、応答をREST APIで返す実装方法を調査する。

## 調査結果

### 1. 必要な依存関係

Spring BootプロジェクトでSpring AIとBedrockを使用するために必要な依存関係：

```gradle
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    implementation 'org.springframework.ai:spring-ai-bedrock-ai-spring-boot-starter'
    // AWS SDK for Java v2も必要
    implementation 'software.amazon.awssdk:bedrock-runtime'
}
```

### 2. 設定ファイル（application.yml）

```yaml
spring:
  ai:
    bedrock:
      aws:
        region: ap-northeast-1  # 使用するAWSリージョン
        access-key: ${AWS_ACCESS_KEY_ID}
        secret-key: ${AWS_SECRET_ACCESS_KEY}
      anthropic:
        claude:
          model: anthropic.claude-3-sonnet-20240229-v1:0
          options:
            temperature: 0.7
            max-tokens: 1000
```

### 3. 実装アーキテクチャ

#### Controller層
- POSTリクエストを受け取る
- リクエストボディからプロンプトを取得
- Serviceクラスに処理を委譲
- レスポンスを返す

#### Service層
- Spring AIのChatClientを使用
- Bedrockへのリクエスト送信
- レスポンスの処理

### 4. 実装に必要なクラス

#### BedrockController
- `@RestController`アノテーション
- `@PostMapping("/")`でルートパスのPOSTを処理
- リクエストボディを受け取るDTOクラス

#### BedrockService
- `@Service`アノテーション
- `ChatClient`を注入
- Bedrockとの通信処理

#### リクエスト/レスポンスDTO
- リクエスト用：プロンプトを含む
- レスポンス用：生成されたテキストを含む

### 5. Spring AIのChatClient使用方法

```java
@Autowired
private ChatClient chatClient;

public String generateResponse(String prompt) {
    return chatClient.prompt()
        .user(prompt)
        .call()
        .content();
}
```

### 6. AWS認証設定

以下のいずれかの方法でAWS認証を設定：

1. **環境変数**
   - `AWS_ACCESS_KEY_ID`
   - `AWS_SECRET_ACCESS_KEY`
   - `AWS_REGION`

2. **IAMロール**（EC2やECS上で実行する場合）

3. **AWS Credentials File**
   - `~/.aws/credentials`
